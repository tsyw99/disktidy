# DiskTidy 前端样式开发规范

---

## 1. 技术架构要求

### 1.1 智能体协作规范

- 所有前端样式开发工作必须通过调用"前端大师"智能体完成
- 智能体在开发过程中需遵循本文档的所有规范要求
- 开发完成后需进行代码审查，确保符合规范标准

### 1.2 组件库融合要求

- 前端界面实现需深度融合 reactbits 组件库，确保组件的充分利用和最佳实践
- 优先使用 reactbits 组件库中的现有组件，避免重复造轮子
- 在使用 reactbits 组件时，需根据项目需求进行适当的样式定制

### 1.3 技术栈规范

| 技术          | 版本   | 用途                         |
| ------------- | ------ | ---------------------------- |
| React         | 18.x   | 前端框架                     |
| TypeScript    | 5.x    | 类型安全                     |
| Tailwind CSS  | 3.x    | 样式解决方案（动画效果核心） |
| Ant Design    | 5.x    | UI 组件库（需定制化）        |
| ECharts       | 5.x    | 数据可视化                   |
| Framer Motion | latest | 动画库（reactbits 依赖）     |
| GSAP          | latest | 高级动画（reactbits 依赖）   |

## 2. 导航组件规范

### 2.1 Dock 导航组件

应用导航系统必须使用 reactbits 组件库中的 Dock 组件实现。

#### 2.1.1 Dock 组件配置规范

```typescript
interface DockItem {
  icon: React.ReactNode;
  label: string;
  onClick: () => void;
  className?: string;
}

interface DockConfig {
  items: DockItem[];
  spring: { mass: number; stiffness: number; damping: number };
  magnification: number; // 放大倍数，默认 70
  distance: number; // 影响距离，默认 200
  panelHeight: number; // 面板高度，默认 68
  dockHeight: number; // Dock 高度，默认 256
  baseItemSize: number; // 基础图标尺寸，默认 50
}
```

#### 2.1.2 导航设计原则

- **功能性**：导航项需清晰标识功能，支持快捷操作
- **视觉美感**：采用 macOS 风格的放大效果，鼠标悬停时图标平滑放大
- **用户体验**：支持键盘导航，具备完善的焦点管理
- **响应式**：在小屏幕设备上自动调整布局和尺寸

#### 2.1.3 导航项设计规范

| 导航项     | 图标建议      | 功能说明           |
| ---------- | ------------- | ------------------ |
| 系统概览   | 仪表盘图标    | 首页，展示系统信息 |
| 磁盘扫描   | 搜索/扫描图标 | 磁盘扫描功能       |
| 垃圾清理   | 垃圾桶图标    | 垃圾文件清理       |
| 大文件分析 | 文件图标      | 大文件分析         |
| 设置       | 齿轮图标      | 应用设置           |

## 4. 动画效果规范

### 4.1 Tailwind CSS 动画核心规范

所有页面动画效果必须基于 Tailwind CSS 实现，确保性能和一致性。

#### 4.1.1 基础动画类

```css
/* 淡入淡出 */
.animate-fade-in {
  animation: fadeIn 0.3s ease-out;
}
.animate-fade-out {
  animation: fadeOut 0.3s ease-out;
}

/* 滑动 */
.animate-slide-up {
  animation: slideUp 0.4s ease-out;
}
.animate-slide-down {
  animation: slideDown 0.4s ease-out;
}
.animate-slide-left {
  animation: slideLeft 0.4s ease-out;
}
.animate-slide-right {
  animation: slideRight 0.4s ease-out;
}

/* 缩放 */
.animate-scale-in {
  animation: scaleIn 0.2s ease-out;
}
.animate-scale-out {
  animation: scaleOut 0.2s ease-out;
}

/* 弹跳 */
.animate-bounce-in {
  animation: bounceIn 0.5s ease-out;
}

/* 脉冲 */
.animate-pulse-soft {
  animation: pulseSoft 2s ease-in-out infinite;
}
```

#### 4.1.2 过渡效果规范

```css
/* 快速过渡 - 150ms */
.transition-fast {
  transition: all 150ms ease;
}

/* 标准过渡 - 200ms */
.transition-normal {
  transition: all 200ms ease;
}

/* 慢速过渡 - 300ms */
.transition-slow {
  transition: all 300ms ease;
}

/* 弹性过渡 */
.transition-bounce {
  transition: all 300ms cubic-bezier(0.68, -0.55, 0.265, 1.55);
}
```

### 4.2 性能要求

- **帧率要求**：所有动画必须保持 60fps，避免卡顿
- **GPU 加速**：使用 `transform` 和 `opacity` 属性触发 GPU 加速
- **避免重排**：动画过程中避免改变 `width`、`height`、`margin` 等属性
- **合理使用 `will-change`**：仅在必要时使用，避免过度使用

### 4.3 动画一致性

| 交互类型   | 动画效果            | 持续时间      |
| ---------- | ------------------- | ------------- |
| 按钮悬停   | 轻微上浮 + 阴影增强 | 200ms         |
| 卡片悬停   | 边框发光 + 轻微放大 | 300ms         |
| 模态框打开 | 淡入 + 缩放         | 200ms         |
| 页面切换   | 淡入淡出            | 300ms         |
| 列表项进入 | 交错滑入            | 每项延迟 50ms |
| 加载状态   | 脉冲旋转            | 持续循环      |

### 4.4 reactbits 动画组件使用

| 组件            | 用途     | 性能建议           |
| --------------- | -------- | ------------------ |
| FadeContent     | 内容渐显 | 适用于页面初始加载 |
| AnimatedContent | 内容动画 | 配合滚动触发使用   |
| ScrollReveal    | 滚动显示 | 长列表优化渲染     |
| ClickSpark      | 点击特效 | 按钮点击反馈       |

## 5. 组件开发规范

### 5.1 组件选择原则

智能体在页面开发过程中，需按以下优先级选择组件：

1. **reactbits 组件库**：优先使用现有组件
2. **Ant Design 组件**：次优先，需定制/修改样式
3. **自定义组件**：仅在无法满足需求时开发

### 5.2 组件设计原则

- **统一设计风格**：所有组件遵循统一的视觉语言
- **一致性交互**：相同类型的交互行为保持一致
- **现代化标准**：符合当前主流设计趋势
- **可维护性**：代码结构清晰，易于理解和修改
- **可扩展性**：预留扩展接口，支持未来功能增强

### 5.3 组件命名规范

```
组件命名：PascalCase（如：DiskCard, ScanProgress）
样式文件：组件名.css（如：DiskCard.css）
类型定义：组件名 + Props（如：DiskCardProps）
```

### 5.4 组件目录结构

```
src/
├── components/
│   ├── common/           # 通用组件
│   │   ├── Modal/
│   │   ├── Toast/
│   │   └── Form/
│   ├── layout/           # 布局组件
│   │   ├── Header/
│   │   ├── Sidebar/
│   │   └── Footer/
│   └── business/         # 业务组件
│       ├── DiskCard/
│       ├── ScanProgress/
│       └── CleanResult/
```

## 8. 整体风格要求

### 8.1 设计理念

- **差异化**：独特的视觉风格，避免与常见工具类软件雷同
- **现代化**：采用当前主流设计趋势，如玻璃拟态、渐变、微交互
- **美观性**：精心设计的视觉层次和配色方案
- **流畅性**：丝滑的动画过渡和交互反馈
- **专业性**：符合磁盘清理工具的专业定位

### 8.2 视觉风格定位

**关键词**：科技感、专业、高效、现代、优雅

**风格参考**：

- 深色主题为主，营造专业氛围
- 紫色渐变作为主色调，体现科技感
- 玻璃拟态效果，增加层次感
- 微妙的光效和阴影，提升质感

## 19. 性能优化

### 19.1 CSS 优化

- 使用 CSS 变量减少重复代码
- 避免深层嵌套选择器
- 合理使用 `will-change` 属性

### 19.2 动画优化

- 优先使用 `transform` 和 `opacity`
- 避免动画过程中触发重排
- 使用 `requestAnimationFrame` 处理复杂动画

### 19.3 渲染优化

- 使用虚拟滚动处理长列表
- 图片懒加载
- 组件按需加载
